What is NSL-KDD?

NSL-KDD is the improved version of the KDD Cup 1999 dataset, created by the Canadian Institute for Cybersecurity. It removes duplicate records from the original KDD dataset, making it more representative for training and evaluating ML-based intrusion detection systems. It is one of the most widely cited datasets in academic NIDS research.

Feature Importances — What the Model Prioritises

The Random Forest assigns importance scores to each feature based on how much they reduce impurity (Gini index) across all 100 trees.

Features like src_bytes, count, and serror_rate are top predictors because they differ dramatically between attack types and normal traffic.

The rule-based system only uses 3 features — missing most of this signal.

AI vs Rule-Based Detection
Why machine learning outperforms manual threshold rules
Rule-Based System — How It Works and Where It Fails
IF count > 400 AND serror_rate > 0.85 → DoS
Detects only extreme SYN-flood attacks. Threshold is too high.
Misses: Moderate DoS (count 150–400), UDP floods, slow-rate attacks.
ELSE IF diff_srv_rate > 0.85 AND rerror_rate > 0.80 → PortScan
Catches only aggressive full-port scans — stealthy scans evade it.
Misses: Slow scans (diff_srv 0.60–0.85), targeted service scans, SYN scans.
ELSE IF num_failed_logins > 5 AND logged_in == 0 → BruteForce
Only fires after 5+ failed logins — attacker may succeed before this triggers.
Misses: Early-stage brute force (1–5 attempts), distributed BF attacks.
ELSE → Normal
Everything not caught above is labelled safe — a large false negative pool.
This is where most missed attacks accumulate.
